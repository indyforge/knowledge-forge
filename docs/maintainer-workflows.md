# Maintainer Workflows

Knowledge Forge is designed for recurring maintainer context work.

## Issue Triage

Capture linked source material, preserve excerpts and hashes, then prepare a brief that separates evidence from unanswered questions.

## Pull Request Review

Collect the PR description, linked issues, migration notes, and benchmark claims before review. Store the context as Markdown so the reasoning can be revisited later.

## Release Review

Compare changes, tests, docs, and known risks before tagging a release. Keep release notes tied to source artifacts and verification output.

## Agent-Assisted Work

Use Codex or another agent to draft summaries, test gaps, or PR descriptions, but save the resulting artifact with provenance and evidence gaps.
