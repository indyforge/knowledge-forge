# Security Policy

## Supported Versions

The public project is currently pre-1.0. Security fixes target the latest `main` branch until versioned releases are established.

## Data Boundaries

Knowledge Forge is local-first. It should not upload source material, notes, manifests, or artifacts unless a future feature explicitly documents and asks for that behavior.

Do not commit:

- API keys,
- cookies,
- personal vault paths,
- private notes,
- local database contents,
- or captures that are not safe to publish.

## Reporting Security Issues

Open a private security advisory on GitHub if available, or contact the maintainer through the GitHub profile. Please include a minimal reproduction and avoid sharing private source material.
